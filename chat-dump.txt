Power Button as Screen Toggle

Two pieces: tell logind to not handle the power button, then let KDE Powerdevil handle it as "turn off screen."
nix

# configuration.nix
services.logind = {
  powerKey = "ignore";        # don't suspend/poweroff
  powerKeyLongPress = "poweroff"; # long-press as safety escape
};

Then in Plasma Manager (or ~/.config/powerdevilrc via home-manager):
nix

# home-manager / plasma-manager
programs.plasma.powerdevil = {
  AC = {
    powerButtonAction = "turnOffScreen";
    dimDisplay.enable = false;
    turnOffDisplay.idleTimeout = "never"; # or a value you like
    autoSuspend.action = "nothing";
  };
};

This gives you: power button press toggles DPMS off, any input (mouse/keyboard) wakes the displays back up. Boot defaults to screens on since DPMS starts in "on" state.

If you're on Wayland, the DPMS wake-on-input is handled by KWin natively. On X11, make sure DPMS is enabled:
nix

services.xserver.displayManager.setupCommands = ''
  ${pkgs.xorg.xset}/bin/xset dpms 0 0 0   # no auto-timeout
  ${pkgs.xorg.xset}/bin/xset +dpms          # but DPMS enabled for manual toggle
'';

Resilient Zellij Session via systemd User Service

Zellij has a client-server architecture. The server survives client disconnects. The trick is to start the server outside the Plasma session graph so it doesn't get killed when Plasma crashes.
nix

# home-manager
systemd.user.services.zellij-server = {
  Unit = {
    Description = "Zellij server daemon";
    # Start early, independent of graphical-session
    After = [ "default.target" ];
  };
  Service = {
    Type = "simple";
    # Start a named session server
    ExecStart = "${pkgs.zellij}/bin/zellij --session main setup --generate-auto-start 2>/dev/null; ${pkgs.zellij}/bin/zellij --session main attach --create";
    # Actually, zellij doesn't have a pure "server-only" mode easily.
    # Better approach: just ensure the server is spawned
    Restart = "on-failure";
  };
  Install.WantedBy = [ "default.target" ];
};

Actually, Zellij's server spawns implicitly on first attach --create. A cleaner approach: don't run a systemd service for the server — instead just make your terminal auto-attach, and rely on Zellij's server persisting after client disconnect:
nix

# In your shell config (e.g. programs.zsh or programs.bash via home-manager)
programs.zsh.initExtra = ''
  # Auto-attach to Zellij if inside a graphical terminal (not SSH, not already in Zellij)
  if [[ -z "$ZELLIJ" && -z "$SSH_CONNECTION" && "$TERM_PROGRAM" != "" ]]; then
    exec ${pkgs.zellij}/bin/zellij attach --create main
  fi
'';

This way:

    First terminal launch spawns the Zellij server + creates session "main"
    Plasma crashes → KWin/Konsole die → Zellij client dies, but the server process (and all child shells) survive because the server is reparented to PID 1
    Plasma restarts → you open a terminal → shell auto-runs zellij attach main → reconnects to the surviving session with all your panes/tabs intact

The server survives because it forks off and isn't a direct child of the terminal emulator's process tree. The shell processes inside Zellij are children of the Zellij server, not of Konsole.

One caveat: if SDDM restarts your entire user session (full logout + re-login), systemd will kill all user-scope processes via user@.service teardown. A pure Plasma crash without SDDM restart is fine. If you want to survive even that, you'd need lingering:
nix

users.users.stefan = {
  linger = true;  # keeps user systemd instance alive even when not logged in
};

This ensures the Zellij server (and any user services) persist regardless of session state.

# home-manager: e.g. home.file or a package overlay
# ~/.local/bin/claude-sandbox

home.file.".local/bin/claude-sandbox" = {
  executable = true;
  text = ''
    #!/usr/bin/env bash
    set -euo pipefail

    PROJECT_DIR="$(realpath "''${1:-.}")"
    PROJECT_NAME="$(basename "$PROJECT_DIR")"
    SANDBOX_HOME="/home/stefan"

    mkdir -p "$GLOBAL_CLAUDE"

      --die-with-parent \
      \
      /* base filesystem (read-only) */ \
      --ro-bind /nix/store /nix/store \
      --ro-bind /nix/var /nix/var \
      --ro-bind /etc /etc \
      --ro-bind /usr /usr \
      --ro-bind /run/current-system /run/current-system \
      --proc /proc \
      --dev /dev \
      --tmpfs /tmp \
      \
      /* project directory (read-write) */ \
      --bind "$PROJECT_DIR" "$SANDBOX_HOME/project" \
      \
      /* global claude config (read-only) */ \
      --ro-bind "$GLOBAL_CLAUDE/CLAUDE.md" "$SANDBOX_HOME/project/CLAUDE.md" \
      --ro-bind "$GLOBAL_CLAUDE/skills" "$SANDBOX_HOME/.claude/skills" \

      --bind "$SANDBOX_TMP" /tmp \
      --tmpfs "$SANDBOX_HOME/.cache" \

Cleaner: As a Nix Package
nix

# packages/claude-sandbox.nix
{ writeShellScriptBin, bubblewrap, ... }:

writeShellScriptBin "claude-sandbox" ''
  set -euo pipefail

  PROJECT_DIR="$(realpath "''${1:-.}")"
  PROJECT_NAME="$(basename "$PROJECT_DIR")"
  GLOBAL_CLAUDE="$HOME/.claude-global"
  SANDBOX_TMP="/tmp/claude-sandbox-$PROJECT_NAME"
  SANDBOX_HOME="/home/$(whoami)"

  mkdir -p "$SANDBOX_TMP"

  # Collect optional bind mounts
  EXTRA_ARGS=()

  # Overlay global CLAUDE.md into project root (read-only)
  # Only if project doesn't have its own
  if [[ -f "$GLOBAL_CLAUDE/CLAUDE.md" && ! -f "$PROJECT_DIR/CLAUDE.md" ]]; then
    EXTRA_ARGS+=(--ro-bind "$GLOBAL_CLAUDE/CLAUDE.md" "$SANDBOX_HOME/project/CLAUDE.md")
  fi

  # Expose global skills directory
  if [[ -d "$GLOBAL_CLAUDE/skills" ]]; then
    EXTRA_ARGS+=(--ro-bind "$GLOBAL_CLAUDE/skills" "$SANDBOX_HOME/project/.claude/skills")
  fi

  # Optional: pass through SSH agent for git
  if [[ -n "''${SSH_AUTH_SOCK:-}" ]]; then
    EXTRA_ARGS+=(--ro-bind "$SSH_AUTH_SOCK" "$SSH_AUTH_SOCK"
                 --setenv SSH_AUTH_SOCK "$SSH_AUTH_SOCK")
  fi

  exec ${bubblewrap}/bin/bwrap \
    --die-with-parent \
    --unshare-pid \
    --share-net \
    \
    --ro-bind /nix /nix \
    --ro-bind /etc /etc \
    --ro-bind /run/current-system /run/current-system \
    --ro-bind /run/opengl-driver /run/opengl-driver \
    --proc /proc \
    --dev /dev \
    \
    --bind "$PROJECT_DIR" "$SANDBOX_HOME/project" \
    --bind "$SANDBOX_TMP" /tmp \
    --tmpfs "$SANDBOX_HOME/.cache" \
    \
    --ro-bind "$HOME/.claude.json" "$SANDBOX_HOME/.claude.json" \
    --ro-bind "$HOME/.gitconfig" "$SANDBOX_HOME/.gitconfig" \
    "''${EXTRA_ARGS[@]}" \
    \
    --setenv HOME "$SANDBOX_HOME" \
    --setenv NIX_PATH "nixpkgs=/nix/var/nix/profiles/per-user/root/channels/nixos" \
    --chdir "$SANDBOX_HOME/project" \
    \
    /run/current-system/sw/bin/bash -lc \
      'nix-shell --run "claude --dangerously-skip-permissions"'
''

```
## Global Files Layout
```

The --ro-bind overlay logic means:

    Project has its own CLAUDE.md → global one is NOT injected (project's takes precedence)
    Project has no CLAUDE.md → global one appears at the project root, read-only
    Skills always appear at .claude/skills/ inside the sandbox

Usage with Zellij

Combine with the Zellij setup from before — each project gets its own Zellij session:
bash

# ~/.local/bin/dev
#!/usr/bin/env bash
PROJECT_DIR="$(realpath "${1:-.}")"
PROJECT_NAME="$(basename "$PROJECT_DIR")"

zellij attach --create "dev-$PROJECT_NAME" \
  options --default-shell "claude-sandbox $PROJECT_DIR"

So dev ~/projects/foo opens a Zellij session named dev-foo that runs the sandboxed Claude inside it. Plasma can crash, you reconnect to the Zellij session, Claude is still running.
What This Isolates
Resource	Access
Project files	read-write
Other projects	invisible
$HOME dotfiles	invisible (only explicit passthrough)
Nix store	read-only
Network	unrestricted
PID namespace	isolated (can't signal other projects)
/tmp	per-project
Global Claude config	read-only overlay
Caveats

    Nix store writes: nix-shell may need to build/fetch derivations. If so, you'll need --bind /nix/store /nix/store (read-write) and --bind /nix/var /nix/var. Alternatively, pre-build the shell outside the sandbox: nix-shell "$PROJECT_DIR/shell.nix" --run true before entering bwrap with a read-only /nix.
    /run/nix: the Nix daemon socket lives at /run/nix/nix-daemon.sock — you need --ro-bind /run/nix /run/nix if the sandbox needs to call nix-build or similar.
    Claude Code's Node: make sure node is either in the shell.nix or globally available via the bind-mounted Nix store. Claude Code itself needs to be accessible too — either in shell.nix or bind-mount its install location.
